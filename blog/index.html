JavaScript模块化

2023年1月6日 · 阅读需 11 分钟

1.模块化入门​

1.1什么是模块化​

  • 将一个复杂的程序依据一定的规则(规范)拆成几个块(文件), 并进行组合在一起。
  • 块的 内部数据/实现 是私有的, 只是向外部暴露一些接口(方法)与外部其它模块通信。
自己的理解：形象的说就是将一个大的JavaScript文件(具有多个功能)，拆封成多个小的JavaScript文件(只有一个或少许几个功能)，在使用这些文件时，再将它们组合在一起。

1.2为什么要模块化​

  • 降低复杂度，提高解耦性
  • 避免命名冲突
  • 更好的分离，按需加载
  • 更高复用性，高可维护性

1.3模块化概念所带来的问题​

  • 请求过多
  • 依赖模糊
  • 难以维护

2.模块化规范​

前言：一个大的项目必定会使用模块化技术，使用模块化就会使用相应的模块化规范，现在比较流行的规范有以下几种：CommonJS、ES6

2.1CommonJS​

2.1.1规范​

  1. 每个文件（js）都可当作一个模块
  2. CommonJS模块化的代码既可以在服务端运行，也可以在浏览器端运行
  3. 在服务器端：模块化的代码可直接运行------------------------Node.js
  4. 在浏览器端：模块化的代码要经过Browserify编译---------Browserify (编译器：将require变为浏览器可以认识的东西)

2.1.2基本语法​

  1. 暴露模块语法

    • 第一种方式：module.exports = value
    • 第二种方式：exports.xxx = value

  2. 引入模块语法

    • 引入第三方模块：require(xxx), xxx为模块名
    • 引入自定义模块：require(xxx),xxx为模块文件路径

  3. 备注：内置关系

    module.exports和exports都指向同一个地址，初始化时module.exports === exports为true

    暴露的本质是module.exports的内容：不管什么时候，暴露的内容永远是module.exports指向的内容

2.1.3 CommonJS在服务器端运行示例​

①.module.exports = value暴露方式和引入自定义模块

​ 📄1_CommonJS_Node\module1.js文件进行暴露

/* 
    module1使用module.exports = xxxx 去暴露，xxx就是暴露的内容
    module.exports和exports不能混用，若混用了，暴露的是module.exports中的内容
*/
//没有暴露的data和msg(被保护了)
const data = 'atguigu'
const msg = 'hello'

module.exports = {
    showData (){
        console.log(data);
    },
    showMsg(){
        console.log(msg);
    }
}

/*
    此行代码不会暴露出去。
    因为上方修改了module.exports，module.exports === exports为false,
    暴露的内容永远是module.exports指向的内容。
*/
exports.x = 100

​ 📄1_CommonJS_Node\app.js文件引入自定义模块

//暴露的本质是module.exports的内容
// 引入的内容是什么，取决于暴露的是什么
const m1 = require('./module1') //引入自定义模块
m1.showData()
m1.showMsg()

//运行方式：右键1_CommonJS_Node文件，点击‘在集成终端中打开’，在终端中输入“node app”
//结果：atguigu （回车）  hello

②.exports.xxx = value暴露方式和引入自定义模块

​ 📄1_CommonJS_Node\module2.js文件进行暴露

/* 
    module2使用exports.xxxxx = value 去暴露，value就是暴露的内容,xxxxx是他的名字
*/

exports.data  = 'atguigu2'
exports.msg = 'hello2'

exports.sum = function (a,b){
    console.log(a+b);
}

exports.sub = function (a,b){
    console.log(a-b);
}

​ 📄1_CommonJS_Node\app.js文件引入自定义模块

const m2 = require('./module2') //引入自定义模块
m2.sum(1,2)
m2.sub(3,4)
//结果 3 (回车) -1

③引入第三方模块

📄1_CommonJS_Node\app.js文件引入第三方模块

/* 
    引入第三方模块：
    1.npm init --yes （nodejs初始化，生成的默认的package.json）
        package.json包含内容：
            version 版本号，一般从0.1.0开始
            description描述
            keywords 关键词用逗号隔开
            author名字加邮箱
            license UNLICENSE，一般不授权公用
    2.npm install uniq --save（安装uniq）
    3.使用第三方模块
*/

const uniq = require('uniq')//引入第三方模块

const arr = [1,3,3,4,2,5]
//去重，并按照从小到大排序（在写代码时一般不使用。数组的方法使用lodash）
console.log(uniq(arr));

2.1.4 CommonJS在浏览器端运行​

  1. 全局安装Browserify==> npm install -g browserify

  2. 编译指定文件

    #右键2_CommonJS_Browserify文件，点击‘在集成终端中打开’，在终端中输入
    browserify app.js -o build.js
    browserify <引入模块化的文件> -o(输出到) <html引入的js文件>

  3. html页面中引入build.js文件

2.2 ES6模块化规范​

2.2.1 规范​

  1. 每个js文件都是一个模块
  2. 要借助 Babel 和 Browserify 依次编译代码，才能在浏览器端运行。
  3. Babel 中文网：https://babel.docschina.org/

2.2.2 基本语法​

  1. 暴露语法：

    • 分别暴露：export 暴露内容
    • 统一暴露：export 暴露内容2
    • 默认暴露：export default 暴露内容

  2. 引入模块：

    • 方法一：import {xxx,yyy} from './module1'
    • 方法二：import module2 from './module2'

  3. 使用规则：

    • 若使用分别暴露、统一暴露的方式暴露内容，那么就用方法一引入
    • 若使用默认暴露的方式暴露内容，那么就用方法二引入

  4. 创建文件结构和准备相关的依赖包

    • 创建文件结构如下：

      -3_ES6_modular
          -src
              -app.js
              -module1.js
          -index.html
          -.babelrc

    • 全局安装 babel-cli（babel命令行的一个工具）、Browserify，执行命令如下：

      #Browserify已在使用CommonJS时已安装
      npm install babel-cli -g

    • 局部安装 babel-preset-env，执行命令如下：

      #在2017年第三季度我们终于要和 ES2015 preset 说再见了
      #npm install babel-preset-es2015 --->不再使用
      #"babel-preset-env": "1.7.0"
      npm install --save-dev babel-preset-env@1.7.0#Babel 6版本写法

    • 定义 .babelrc 文件

      #Babel 6.x:Preset:- babel-preset-env --->Babel 6.x版本写法
      npm install --save-dev babel-preset-env@1.7.0
      { "presets": [ "env" ],}

      #未实践
      # Babel 7.x:Preset:- @babel/preset-env --->Babel 7.x版本写法
      npm install --save-dev @babel/preset-env
      { "presets": ["@babel/preset-env"] }

  5. 编译代码：

    • 进入 "3_ES6_modular" 目录，使用Babel将ES6编译为ES5语法

      #src文件夹中存放使用ES6语法的文件(主要是import export)
      #生成的build文件夹中的文件是将src中的文件的ES6写法转换成CommonJS写法
      babel ./src -d ./build

    • 继续使用Browserify编译上一步的js

      browserify ./build/app.js -o ./build/build.js
    • 当原文件修改时，需要重新用命令进行编译
    • 备注：命令不要记！！我们学习的是模块化的暴露、引入语法，不是命令，以后也不敲命令。

  6. 在页面中引入build.js进行测试

2.2.3 ES6模块化示例​

  1. 分别暴露和引入

    • 分别暴露

      //module1中使用【分别暴露】
      export const data = 'atguigu'
      export const msg = 'hello,0826'

      export function showData(){
          console.log(data);
      }
      export function showMsg(){
          console.log(msg);
      }

      //module2中使用【分别暴露】，但暴露了一个和module1重名的一个data
      export const data = '北七家镇宏福科技园'

    • 分别暴露的引入(app.js)

      //引入【分别暴露】的模块
      //{data,showData,showMsg} 不是解构赋值，只是ES6的一种写法
      import {data,showData,showMsg} from './module1'

      //引入【分别暴露】的模块+重命名
      import {data as data2} from './module2'

      //引入【分别暴露】的模块+打包引入
      import * as module1 from './module1'
      //此时module1为一个对象
      //可以进行解构赋值，key为定义的变量、方法名；value为变量值、function
      const {data,msg} = module1

  2. 统一暴露

    • 统一暴露

      /* 
          module3中使用【统一暴露】
      */
      const school = '尚硅谷'
      const person = {
          name:'老刘',
          age:19,
          sex:'女'
      }
      function getLaoliu(){
          console.log(person);
      }

      //统一暴露(精简版)---用的多
      export {school,person,getLaoliu}

      //统一暴露(完整版)---起别名后，引入时要是用新名字
      // export {school as a,person as b,getLaoliu as c}

    • 统一暴露的引入(app.js)

      //引入【统一暴露】的模块（统一暴露和分别暴露，都可以使用上方三种引入的方式）
      import {school,person,getLaoliu} from './module3'

  3. 默认暴露

    • 默认暴露

      /* 
          module4中使用【默认暴露】-----一个文件中只能暴露一次！！！！
      */
      export default {
          name:"wc",
          age:5,
      }

    • 默认暴露的引入

      //引入【默认暴露】的模块
      //module4 不能写为{name,age}---->这样会认为去module4文件中找export{}
      import module4 from './module4'

  4. 多种暴露方式的使用

    • 多种暴露方式

      /* 混合使用暴露方式 */
      //使用【分别暴露】
      export const teacher1 = {name:'强哥',age:15}
      export const teacher2 = {name:'歌神',age:17}

      //使用【统一暴露】
      const stu1 = {name:'王宇',age:18}
      const stu2 = {name:'宇航',age:19}
      export {stu1,stu2}

      //使用【默认暴露】
      export default {
          school:'尚硅谷',
          address:'宏福科技园',
          subjects:['前端','java','大数据']
      }

    • 引入多种暴露方式的模块

      import module5,{teacher1,teacher2,stu1,stu2} from './module5'
      console.log(module5);//默认暴露中的内容{school:'尚硅谷',address:'宏福科技园',..}
      console.log(teacher1);//{name:'强哥',age:15}
      console.log(teacher2);//{name:'歌神',age:17}
      console.log(stu1);//{name:'王宇',age:18}
      console.log(stu2);//{name:'宇航',age:19}
